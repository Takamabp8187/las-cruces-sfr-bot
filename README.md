# 🏠 Las Cruces SFR Daily Report Bot

Automatically searches Zillow, Redfin, and Realtor.com every morning for
single-family homes in Las Cruces, NM under $180,000 with ≥2 bedrooms,
≥1 bathroom, and ≥1,000 sq ft — then emails you the results.

**Day 1:** All active qualifying listings (baseline)  
**Day 2+:** Only new listings and price changes since yesterday

---

## 🚀 Setup (GitHub Actions — Free, No Server Needed)

### Step 1 — Fork or upload this repo to GitHub

1. Go to [github.com](https://github.com) and create a free account if needed.
2. Click **"New repository"** → name it `las-cruces-sfr-bot` → **Create**.
3. Upload all files from this folder into the repo (drag-and-drop on GitHub.com works).

### Step 2 — Create a Gmail App Password

> You need this so the script can send email on your behalf without
> storing your real password.

1. Go to [myaccount.google.com/security](https://myaccount.google.com/security)
2. Enable **2-Step Verification** if not already on.
3. Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
4. Select app: **Mail** · device: **Other** → name it `sfr-bot` → **Generate**
5. Copy the 16-character password shown (you won't see it again).

### Step 3 — Add Secrets to GitHub

In your GitHub repo, go to **Settings → Secrets and variables → Actions → New repository secret** and add:

| Secret name        | Value                              |
|--------------------|-------------------------------------|
| `GMAIL_USER`       | Your Gmail address (sender)         |
| `GMAIL_APP_PASS`   | The 16-char App Password from Step 2|
| `RECIPIENT_EMAIL`  | `bartljordan@gmail.com`             |

### Step 4 — Enable GitHub Actions

1. In your repo, click the **Actions** tab.
2. If prompted, click **"I understand my workflows, go ahead and enable them."**
3. The workflow is now scheduled to run daily at **8:00 AM Mountain Time**.

### Step 5 — Test it manually

1. Go to **Actions → Daily Las Cruces SFR Report**
2. Click **"Run workflow"** → **"Run workflow"**
3. Watch it run and check your inbox!

## 📝 Notes

- **Anti-scraping:** Zillow and Realtor.com may occasionally block the
  script. If you get no Zillow results for several days in a row, consider
  using a proxy service like [ScraperAPI](https://scraperapi.com) (free tier
  available). Replace the `requests.get(...)` call with:
  ```python
  url = f"http://api.scraperapi.com?api_key=YOUR_KEY&url={url}"
  ```

- **State persistence:** GitHub Actions caches `state.json` between runs.
  If the cache is evicted (after ~7 days of no activity), the next run
  will treat it as Day 1 and send all listings again. This is harmless.

- **MLS data:** These sites pull from the Las Cruces Association of
  REALTORS® MLS. There may be a 15-minute to 24-hour delay on very new
  listings depending on the source.
